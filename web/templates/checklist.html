Migration Checklist

Subscription: {{ subscription_id }}
Overall Progress 0 / 0 completed
0%
Phase 1: Pre-Transfer Actions 0
  • Export RBAC role assignments and custom roles
    All role assignments and custom roles are permanently deleted during transfer. Run tenova export-rbac or use the Export RBAC button on the dashboard.
  • Map source principals to target tenant principals
    Create a principal mapping JSON file that maps old user/group/SP object IDs to their equivalents in the target tenant.
  • Back up Key Vault secrets, certificates, and keys
    Key Vault access policies are wiped during transfer. Export secrets and certificate references before proceeding.
    Phase 2: Execute Transfer
    • Verify readiness check passes (no blockers)
      Run tenova readiness-check or use the Readiness Check on the dashboard. All blockers must be resolved.
    • Schedule maintenance window and notify stakeholders
      Subscription transfer requires downtime. Ensure all teams are aware and a rollback plan is in place.
    • Transfer the subscription to the target tenant
      Use Azure Portal → Subscriptions → Change directory, or run tenova transfer.
    Phase 3: Post-Transfer Actions 0
    • Import RBAC role assignments and custom roles
      Run tenova import-rbac with the exported file and principal mapping.
    • Disable and re-enable system-assigned managed identities
      All system-assigned managed identities are broken after transfer. Disable and re-enable on each resource.
    • Rotate storage account access keys and SAS tokens
      Source tenant users may still have access via access keys. Rotate all keys to revoke access.
      Phase 4: Validation
      • Validate all services are working in the target tenant
        Test application endpoints, verify database connectivity, check Key Vault access, and validate managed identity operations.
      • Run a follow-up scan in the target tenant
        Perform another scan to confirm all impacted resources have been remediated.
      • Close maintenance window and notify stakeholders
        Confirm migration is complete and services are operational.
