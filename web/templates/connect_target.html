Authenticate in Target Tenant

Now that you've scanned the subscription (capturing the "before" state) and transferred it to the target tenant in the Azure portal, sign in with an account in the target (destination) tenant so Azure Sub Migrator can reconfigure resources there using your scan data.

This is a separate sign-in — you won't be signed out of your current session. The target account can be a completely different identity.

The Entra ID (Azure AD) tenant ID of the destination tenant. You can find this in the Azure portal under Entra ID → Overview → Tenant ID.
Workflow & Requirements

Expected workflow:

  1. Scan the subscription in the source tenant (you've done this)
  2. Transfer the subscription to the target tenant via the Azure portal
  3. Reconfigure using this Post-Transfer wizard (you are here)

Before you continue:

  • The subscription must already be transferred to the target tenant via the Azure portal.
  • Your target-tenant account needs Contributor + User Access Administrator roles on the subscription.
  • The target tenant admin must have consented to the Azure Sub Migrator app. Admin consent link
  • RBAC export should be done before the transfer so Azure Sub Migrator has a record of what to recreate.
