Authenticate in Target Tenant

After transferring the subscription, you need to sign in with an account in the target (destination) tenant so Tenova can reconfigure resources there.

This is a separate sign-in — you won't be signed out of your current session. The target account can be a completely different identity.

The Entra ID (Azure AD) tenant ID of the destination tenant. You can find this in the Azure portal under Entra ID → Overview → Tenant ID.
Requirements
  • The subscription must already be transferred to the target tenant via the Azure portal.
  • Your target-tenant account needs Contributor + User Access Administrator roles on the subscription.
  • The target tenant admin must have consented to the Tenova app. Admin consent link
  • RBAC export should be done before the transfer so Tenova can recreate the role assignments.
